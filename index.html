In this webpage, we will explore how to redesign a website to improve its usability and accessibility, from identifying issues to creating Figma prototypes and actually implementing the redesign. We will use the website of Ceremony Cafe, a famous cafe in Providence, Rhode Island, as example üôÇ

What are issues with the Ceremony Cafe website?

Ceremony Cafe in Providence is one of the most famous cafes in the region, so it's interesting to examine whether its online presence is as established as its physical store. Although the website contains all the relevant information, some of it is not organized in a user-friendly manner. The menu page (see scrollable image below) consists of a series of static and hard-to-read images.

Here are some issues with this page:

Accessibility

  • as revealed by WAVE, an automated tool for assessing website accessibility, the alt texts for the menu images are ineffective since they do not capture details about specific drinks
  • for example, the alt text for the bar menu is simply ‚Äúcocktails,‚Äù which is unhelpful to users
  • the small default size of the menu images further lowers accessibility

Learnability

  • there lacks a high-level explanation of the different menus and their display order, which makes it difficult for first-time users to learn the webpage structure
  • the lack of clear differentiation between different types of information (e.g., menu subsection titles, item descriptions) further adds to the confusion

Memorability

  • the menus do not look visually differentiable enough and do not follow a clear color code, so it's easy for users to forget what menus there are when and where they are located, when they revisit the website

Efficiency

  • the page does not effectively support users in finding specific drinks or learning more about them
  • this is because the menus are vertically layered on top of each other, and it takes a long time for users to scroll through the page and find the menu they are interested in

In addition, which doesn't align with user's conceptual models (i.e., common expectations) for cafe websites, where special menus or special items are typically highlighted.

These issues can negatively impact user experience by making it more difficult for users, especially these with disabilities, to access comprehensive information about the drinks on different menus and remember where to find them next time they visit the page. As a result, it may deter potential customers from then ordering items from the cafe or visiting it in-person.

How can we redesign the page?

Let's start by designing a visual style guide for the webpage (see scrollable image below). Inspired by the original website's colors and typography, our style guide assigns distinct color codes to different menus and applies different typography styles to menu sections to create a clear informational hierarchy (e.g. menu title is semi-bolded, item description is light italic).

The primary colors preserve the earthy, calm tone of the original page, while the fresh secondary colors used on buttons help serve as visual indicators for the different menus.

Next, we can create Figma mockups (prototypes) for the redesign. To accommodate different screen sizes, we need to design responsive mockups for desktop (width > 1200px), tablet (600-1200px width), and mobile (width < 600px) users.

As shown in the scrollable desktop mockup below, we added an overall explanation of the menu sections as well as three color-coded buttons that direct users to their desired menu. Also, the menu sections are now built as tables with aria labels for screenreaders, rather than static images, so users with accessibility needs can access the same information as other users.

While the original page includes a long description about the Ceremony tea house, it is removed in the redesigned version since this information is already included on other pages of the website.

In the scrollable tablet and mobile versions below, the number of items per row decreases to reduce information overload but still preserves information completeness. However, styles such as flex box, spacing, and font size remain the same, except for the spacing between buttons and the font size of the nav bar items.

In the mobile version above, the navigation bar turns into an expandable hamburger menu, and the buttons stack on top of each other vertically to maintain a clean and concise layout.

After deploying our redesigned webpage, we ran it through the WAVE tool, which detected no regular or contrast errors (see screenshot below) and confirmed the inclusion of aria labels. This tells us that our updated website is more accessible than before!

Note that the nav bar links in the redesigned page still redirect users to the old Ceremony pages. Since we can't access the original social media icons used in the footer, they have been changed to plain images.

Overall, our redesign addresses the usability and accessibility issues identified in the following ways:

  • Better learnability through clear menu explanations and a logical information hierarchy

  • Better memorability by using color-coded menu buttons to help users recall and navigate between different menus easily

  • Better efficiency by quickly directing users to the menu section they are interested in through the click of menu buttons

  • Better accessibility by making each menu take up bulk of the page space and by adding arial labels for all the drinks in each menu subsection table
